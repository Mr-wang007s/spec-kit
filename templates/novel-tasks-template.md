# 创作任务: [小说名称]

**输入**: 来自 `/specs/[###-小说名称]/` 的设计文档
**前置条件**: novel-plan.md (必需), novel-spec.md (用户故事必需), research.md, characters.md, worldview.md, outline.md

**组织方式**: 任务按情节分组,以实现每个情节的独立创作和阅读。

## 格式: `[ID] [P?] [情节] 描述`

- **[P]**: 可以并行创作(不同章节,无依赖关系)
- **[情节]**: 此任务属于哪个情节(例如: 情节1, 情节2, 情节3)
- 在描述中包含确切的文件路径

- **单一小说**: `manuscript/chapters/`, `manuscript/drafts/` 在仓库根目录

- **多线叙事**: `manuscript/storyline-a/chapters/`, `manuscript/storyline-b/chapters/`

## 依赖关系与执行顺序

### 阶段依赖

- **准备工作(第1阶段)**: 无依赖 - 可以立即开始
- **基础设定(第2阶段)**: 依赖准备工作完成 - 阻塞所有情节创作
- **情节创作(第3阶段+)**: 所有依赖基础设定阶段完成
  - 情节可以并行创作(如果有多人)
  - 或按优先级顺序创作(P1 → P2 → P3)
- **润色(最终阶段)**: 依赖所有期望的情节完成

### 情节依赖

- **情节 1 (P1)**: 可以在基础设定(第2阶段)后开始 - 不依赖其他情节
- **情节 2 (P2)**: 可以在基础设定(第2阶段)后开始 - 可能与情节1整合,但应该独立可读
- **情节 3 (P3)**: 可以在基础设定(第2阶段)后开始 - 可能与情节1/2整合,但应该独立可读

### 每个情节内部

- 大纲先于初稿
- 初稿先于修改
- 核心章节先于衔接章节
- 情节完成后再进入下一个优先级

### 并行机会

- 所有标记为 [P] 的准备工作任务可以并行运行
- 所有标记为 [P] 的基础设定任务可以并行运行(在第2阶段内)
- 一旦基础设定阶段完成,所有情节可以并行开始(如果团队容量允许)
- 一个情节内标记为 [P] 的所有大纲任务可以并行运行
- 不同情节可以由不同创作者并行创作

## 创作策略

### MVP优先(仅情节1)

1. 完成第1阶段: 准备工作
2. 完成第2阶段: 基础设定(关键 - 阻塞所有情节)
3. 完成第3阶段: 情节1
4. **停止并验证**: 独立测试情节1
5. 如果准备好就发布/展示

### 增量交付

1. 完成准备工作 + 基础设定 → 基础就绪
2. 添加情节1 → 独立测试 → 发布/展示(MVP!)
3. 添加情节2 → 独立测试 → 发布/展示
4. 添加情节3 → 独立测试 → 发布/展示
5. 每个情节都增加价值而不破坏之前的情节

### 并行团队策略

有多个创作者时:

1. 团队一起完成准备工作 + 基础设定
2. 一旦基础设定完成:
   - 创作者A: 情节1
   - 创作者B: 情节2
   - 创作者C: 情节3
3. 情节独立完成并整合

## 注意事项

- [P] 任务 = 不同文件,无依赖关系
- [情节] 标签将任务映射到特定情节以便追溯
- 每个情节应该独立可完成和可读
- 在每个检查点停止以独立验证情节
- 避免: 模糊任务、同文件冲突、破坏独立性的跨情节依赖

---

## 质量检查清单

每完成一个情节,检查:

- [ ] 情节逻辑连贯,无明显漏洞
- [ ] 角色行为符合设定,性格一致
- [ ] 对话自然,符合角色身份
- [ ] 场景描写生动,画面感强
- [ ] 节奏把控得当,无拖沓或仓促
- [ ] 文字流畅,无明显语病
- [ ] 伏笔设置合理,有呼应
- [ ] 情感表达到位,能引起共鸣
- [ ] 符合目标读者的阅读习惯
- [ ] 达到预期的字数要求
