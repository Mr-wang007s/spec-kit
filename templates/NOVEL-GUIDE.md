# 小说创作 Spec-Kit 使用指南

欢迎使用 Spec-Kit 小说创作框架!本指南将帮助你使用规格驱动的方式创作小说。

---

## 📚 什么是规格驱动的小说创作?

规格驱动的小说创作是一种结构化的创作方法,它强调:

- **意图驱动**: 先明确"写什么"和"为什么写",再考虑"怎么写"
- **详细规划**: 使用完善的规格说明和创作计划
- **多步骤细化**: 而非一次性从灵感到成稿
- **独立情节**: 每个情节都可以独立创作和阅读
- **增量交付**: 可以逐步发布,而非等待全部完成

## 🚀 快速开始

### 第1步: 建立创作原则

使用 `/novelkit.constitution` 命令创建你的创作原则和指导方针:

```bash
/novelkit.constitution 创建专注于角色深度、情感真实性、节奏控制和读者体验的创作原则
```

这将创建或更新 `.specify/memory/novel-constitution.md` 文件。

### 第2步: 创建小说规格说明

使用 `/novelkit.specify` 命令描述你想写的小说。专注于**写什么**和**为什么写**,而非写作技巧:

```bash
/novelkit.specify 创作一部都市言情小说,讲述一个职场女性在事业和爱情之间寻找平衡的故事。
女主角是一名30岁的广告公司创意总监,事业有成但感情空白。她遇到了两个男人:一个是温柔
体贴的青梅竹马,一个是才华横溢但傲慢的客户。故事探讨现代女性的独立、选择和成长。
重点展现女主角的内心挣扎和最终的自我发现。
```

### 第3步: 创建创作计划

使用 `/novelkit.plan` 命令提供你的创作方法和技术选择:

```bash
/novelkit.plan 采用第一人称视角,以女主角的内心独白为主。文风轻松幽默中带有深度思考。
每章3000-5000字,共30章,预计总字数12万字。采用双时间线叙事,现在和回忆交织。
重点刻画职场细节和情感细腻变化。
```

### 第4步: 分解创作任务

使用 `/novelkit.tasks` 命令从创作计划生成可执行的任务列表:

```bash
/novelkit.tasks
```

### 第5步: 开始创作

使用 `/novelkit.implement` 命令执行创作计划:

```bash
/novelkit.implement
```

## 📖 详细流程

### 步骤1: 建立创作原则 (Constitution)

创作原则是你的创作"宪法",定义了不可违背的准则:

**示例提示**:
```
/novelkit.constitution 创建原则,重点关注:
1. 角色塑造的深度和真实性
2. 情节的逻辑性和吸引力  
3. 文字的可读性和美感
4. 读者体验的优先级
5. 创作质量高于速度
```

**输出**: `.specify/memory/novel-constitution.md`

### 步骤2: 创建规格说明 (Specification)

规格说明定义了你要写的故事的核心内容:

**重要**: 
- ✅ 专注于故事内容、角色、情节、主题
- ❌ 避免指定写作技巧、叙事手法、文学结构

**示例提示**:
```
/novelkit.specify 创作一部科幻悬疑小说《记忆商店》。

故事背景: 2050年,人类可以买卖记忆。主角是记忆商店的老板,专门收购和出售他人的记忆。

主要情节:
1. 主角收购了一段神秘记忆,发现其中隐藏着一桩20年前的谋杀案
2. 随着调查深入,主角发现自己也与这个案件有关
3. 最终揭示真相:主角自己就是凶手,但记忆被删除了

主题: 探讨记忆与身份的关系,如果失去记忆,我们还是原来的自己吗?

目标读者: 喜欢悬疑和科幻的成人读者

预期效果: 让读者在阅读过程中不断猜测,最后的反转令人震撼
```

**输出**: 
- 新分支: `001-记忆商店`
- 规格说明: `specs/001-记忆商店/novel-spec.md`

### 步骤3: 澄清规格说明 (可选但推荐)

在创建计划之前,使用 `/novelkit.clarify` 澄清模糊的地方:

```bash
/novelkit.clarify
```

AI会问一系列问题来澄清不明确的要求,例如:
- 叙事视角是什么?
- 故事的基调是什么?
- 主角的性格特点是什么?
- 结局是开放式还是封闭式?

你也可以主动补充细节:
```
补充说明: 采用第三人称有限视角,跟随主角。基调是悬疑紧张,但不过分黑暗。
主角是一个理性冷静但内心孤独的人。结局是封闭式的,真相完全揭示。
```

### 步骤4: 创建创作计划 (Plan)

创作计划将规格说明转化为具体的创作方法:

**示例提示**:
```
/novelkit.plan 

叙事视角: 第三人称有限,跟随主角
文风: 简洁明快,重视氛围营造
篇幅: 中篇小说,15万字,分为25章
结构: 采用倒叙开头,然后按时间顺序展开,最后回到开头形成闭环
重点: 
- 详细设定记忆交易的世界观
- 精心设计悬疑线索的铺设
- 刻画主角的心理变化
- 准备震撼的结局反转
```

**输出**:
- `specs/001-记忆商店/novel-plan.md` - 创作计划
- `specs/001-记忆商店/research.md` - 背景研究
- `specs/001-记忆商店/characters.md` - 角色设定
- `specs/001-记忆商店/worldview.md` - 世界观设定
- `specs/001-记忆商店/outline.md` - 详细大纲
- `specs/001-记忆商店/writing-guide.md` - 写作指南

### 步骤5: 验证计划

让AI审查创作计划,确保没有遗漏:

```
请审查创作计划和设计文档,检查是否有遗漏或不清楚的地方。
特别关注:
1. 角色设定是否完整
2. 世界观是否自洽
3. 大纲是否覆盖所有主要情节
4. 写作指南是否明确
```

### 步骤6: 生成任务列表

使用 `/novelkit.tasks` 命令生成详细的创作任务:

```bash
/novelkit.tasks
```

**输出**: `specs/001-记忆商店/tasks.md`

任务列表包含:
- 按情节优先级组织的任务
- 每章的创作任务
- 依赖关系和执行顺序
- 可以并行创作的章节标记

### 步骤7: 开始创作

使用 `/novelkit.implement` 命令开始实际创作:

```bash
/novelkit.implement
```

AI将:
- 按照任务列表顺序创作
- 遵循角色设定和世界观
- 保持与写作指南一致
- 提供进度更新

## 🎯 核心概念

### 情节优先级 (P1, P2, P3)

- **P1 (最高优先级)**: 核心情节,构成最小可行故事(MVP)
  - 单独就能成为一个完整的故事
  - 传递核心价值和主题
  - 必须首先完成

- **P2 (中等优先级)**: 重要情节,丰富故事
  - 增加深度和复杂性
  - 但不是核心必需的
  - 在P1完成后创作

- **P3 (较低优先级)**: 补充情节,锦上添花
  - 提供额外的趣味或深度
  - 可以根据时间和篇幅决定是否包含
  - 最后创作

### 情节独立性

每个情节应该:
- 可以独立阅读和理解
- 有完整的起承转合
- 单独就能传递价值
- 不破坏其他情节

**好处**:
- 可以增量发布(先发布P1,再发布P2...)
- 可以并行创作(多人协作)
- 降低风险(P1完成就有可发布的作品)
- 灵活调整(可以根据反馈决定是否继续P2、P3)

### 最小可行故事 (MVP)

最小可行故事是指:
- 只包含P1情节
- 已经是一个完整的故事
- 能够传递核心价值
- 可以独立发布

**示例**:
- 完整故事: 30章,包含主线、感情线、成长线
- MVP (P1): 15章,只包含主线,但有完整的开头、发展、高潮、结局

## 📝 模板说明

### novel-spec-template.md (规格说明模板)

定义**写什么**:
- 故事场景与情节
- 创作要求
- 成功标准
- 边缘情况

**关键点**:
- 使用 [需要澄清] 标记不明确的地方
- 按优先级组织情节
- 每个情节都有独立测试方法

### novel-plan-template.md (创作计划模板)

定义**怎么写**:
- 创作背景(类型、视角、文风等)
- 创作原则检查
- 详细设计(角色、世界观、大纲)
- 阶段划分

**关键点**:
- 通过创作原则检查关卡
- 记录复杂度违规及理由
- 明确里程碑和验收标准

### novel-tasks-template.md (任务列表模板)

定义**执行步骤**:
- 按情节组织的任务
- 依赖关系
- 并行机会
- 质量检查清单

**关键点**:
- 标记可并行任务 [P]
- 标记任务所属情节 [情节1]
- 明确检查点

### novel-checklist-template.md (检查清单模板)

定义**质量标准**:
- 情节完整性
- 角色塑造
- 世界观一致性
- 文字质量
- 叙事技巧

**关键点**:
- 逐项检查
- 记录发现和改进建议
- 追踪完成率

## 🎨 创作示例

### 示例1: 短篇小说

```bash
# 步骤1: 建立原则
/novelkit.constitution 创建原则,强调简洁、情感冲击力和意外结局

# 步骤2: 创建规格说明
/novelkit.specify 创作一个5000字的短篇小说《最后一封信》。
讲述一个老人在生命最后一天给已故妻子写信的故事。
通过信件内容逐渐揭示他们的爱情故事和老人的秘密。

# 步骤3: 创建计划

/novelkit.implement
```

### 示例2: 长篇系列小说

```bash
# 步骤1: 建立原则
/novelkit.constitution 创建原则,强调世界观一致性、角色成长弧光、

# 步骤2: 创建第一部规格说明

第一部主要情节:
P1: 主角适应学院生活,发现自己的特殊天赋

P3: 主角揭露一个威胁学院的阴谋

主题: 成长、友谊、选择
目标: 第一部完整独立,但为后续埋下伏笔

# 步骤3: 创建计划
/novelkit.plan 第三人称有限视角。文风适合青少年读者,轻松中带有深度。
预计30万字,40章。详细设定魔法体系和学院世界观。
重点刻画主角的成长和与同学的关系。

# 后续步骤...
/novelkit.tasks
/novelkit.implement
```

## 💡 最佳实践

### 1. 先规划,后创作

不要急于开始写作,花时间完善:
- 规格说明
- 角色设定
- 世界观
- 详细大纲

**好的规划 = 顺畅的创作**

### 2. 使用优先级

合理分配情节优先级:
- P1: 核心情节,必须有
- P2: 重要情节,最好有
- P3: 补充情节,可以有

**好处**: 即使时间不够,也能完成一个完整的故事(P1)

### 3. 保持情节独立

确保每个情节:
- 可以独立阅读
- 有独立价值
- 不依赖其他情节

**好处**: 灵活调整,增量发布

### 4. 遵循创作原则

定期检查是否遵循创作原则:
- 读者优先
- 简洁性
- 一致性
- 质量优先

**工具**: 使用检查清单

### 5. 迭代改进

不要期望一次写完美:
- 先完成初稿
- 再进行修改
- 多轮润色

**记住**: 好作品是改出来的

### 6. 寻求反馈

在关键节点寻求反馈:
- 规格说明完成后
- 第一个情节完成后
- 初稿完成后

**来源**: 测试读者、写作小组、AI助手

## 🔧 常见问题

### Q1: 我的故事很复杂,超过了5个主要角色怎么办?

**A**: 
1. 重新审视哪些角色真正"主要"
2. 考虑合并相似角色
3. 将一些角色降级为次要角色
4. 如果确实需要,在复杂度跟踪表中说明理由

### Q2: 我想边写边想,不想提前规划怎么办?

**A**: 
规格驱动方法更适合:
- 需要保持一致性的长篇作品
- 多人协作的项目
- 需要增量发布的作品

如果你喜欢自由创作,可以:
- 只使用部分流程(如角色设定、大纲)
- 将规格说明作为指导而非约束
- 在创作过程中逐步完善规格

### Q3: 如何处理创作过程中的灵感变化?

**A**:
1. 更新规格说明文档
2. 评估对现有内容的影响
3. 在复杂度跟踪表中记录变更
4. 重新生成受影响的任务

**记住**: 规格说明是活文档,可以演进

### Q4: 多个情节可以并行创作吗?

**A**:
可以,如果:
- 基础设定阶段已完成
- 情节之间是独立的
- 有多个创作者或足够时间

**建议**: 单人创作时,按优先级顺序更安全

### Q5: 如何确保文风一致?

**A**:
1. 在写作指南中明确文风要求
2. 提供文风示例段落
3. 定期审查已完成章节
4. 使用检查清单验证一致性

## 📚 进阶技巧

### 技巧1: 使用检查清单作为"单元测试"

就像代码有单元测试,小说也可以有:
- 情节逻辑检查清单
- 角色一致性检查清单
- 文字质量检查清单

**每完成一个情节就运行一次检查清单**

### 技巧2: 创建角色档案卡

为每个主要角色创建详细档案:
- 基本信息
- 性格特点
- 背景故事
- 说话方式
- 常用词汇

**写作时随时参考,保持角色一致**

### 技巧3: 绘制情节时间线

特别是对于:
- 多线叙事
- 非线性叙事
- 长时间跨度的故事

**避免时间线混乱**

### 技巧4: 建立灵感库

收集:
- 有趣的对话
- 场景描写灵感
- 情节转折点子
- 角色特质想法

**创作时随时取用**

### 技巧5: 设定每日目标

例如:
- 每天写2000字
- 每周完成1章
- 每月完成1个情节

**保持稳定的创作节奏**
