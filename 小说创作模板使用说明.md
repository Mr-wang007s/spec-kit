# 小说创作模板使用说明

## 📚 欢迎使用 Spec-Kit 小说创作模板!

本项目已经将原本用于软件开发的 Spec-Kit 模板改造成了适合小说创作的中文版本。

## 🎯 这是什么?

这是一套**规格驱动的小说创作方法**,帮助你:

- ✅ 系统化地规划和创作小说
- ✅ 保持角色、情节、世界观的一致性
- ✅ 通过优先级管理情节(P1, P2, P3)
- ✅ 支持增量创作和发布
- ✅ 进行系统化的质量检查
- ✅ 便于团队协作创作

## 🎨 核心模板

### 1. 规格说明书 (`novel-spec-template.md`)

**用途**: 定义你要写的故事

**包含**:
- 故事场景与情节(按优先级P1, P2, P3)
- 创作要求
- 成功标准
- 边缘情况

**何时使用**: 有了故事灵感,想要系统化地定义它

### 2. 创作计划 (`novel-plan-template.md`)

**用途**: 规划如何创作这个故事

**包含**:
- 创作背景(类型、视角、文风等)
- 角色设定
- 世界观设定
- 详细大纲
- 写作指南

**何时使用**: 规格说明完成后,需要详细规划创作方法

### 3. 任务列表 (`novel-tasks-template.md`)

**用途**: 将创作计划分解为具体任务

**包含**:
- 按情节组织的创作任务
- 依赖关系
- 并行创作机会
- 质量检查点

**何时使用**: 创作计划完成后,准备开始实际写作

### 4. 检查清单 (`novel-checklist-template.md`)

**用途**: 系统化地检查作品质量

**包含**:
- 情节完整性(13项)
- 角色塑造(7项)
- 世界观一致性(6项)
- 文字质量(6项)
- 叙事技巧(7项)
- 结构完整性(7项)
- 情感共鸣(5项)
- 主题表达(4项)
- 可读性(5项)
- 创新性(4项)
- 技术规范(5项)
- 最终审查(6项)

**总计**: 68个检查项

**何时使用**: 完成一个情节或整部作品后,进行质量检查

### 5. 创作宪章 (`novel-constitution-template.md`)

**用途**: 建立不可违背的创作原则

**包含**:
- 十条核心原则
  1. 读者优先原则
  2. 简洁性原则
  3. 独立性原则
  4. 一致性原则
  5. 展示优于讲述原则
  6. 冲突驱动原则
  7. 节奏控制原则
  8. 原创性原则
  9. 质量优先原则
  10. 修订与改进原则

**何时使用**: 项目开始时,建立创作的指导原则

## 🎯 核心概念

### 情节优先级

- **P1 (最高)**: 核心情节,构成最小可行故事(MVP)
- **P2 (中等)**: 重要情节,丰富故事深度
- **P3 (较低)**: 补充情节,锦上添花

**好处**: 确保核心故事完整,支持增量发布

### 情节独立性

每个情节应该:
- 可以独立阅读和理解
- 有完整的起承转合
- 单独就能传递价值

**好处**: 可以并行创作,灵活调整,降低风险

### 最小可行故事 (MVP)

只包含P1情节的完整故事,已经可以独立发布。

**好处**: 降低风险,早期获得反馈

```
   ↓

```

## 🎨 快速示例

### 示例: 短篇小说《最后一封信》

#### 1. 规格说明

```markdown
# 小说规格说明书: 最后一封信

## 故事场景与情节

### 情节 1 - 最后的告白 (优先级: P1)

一个老人在生命最后一天给已故妻子写信,通过信件内容
逐渐揭示他们的爱情故事和老人的秘密。

**情节场景**:
1. 场景设定: 老人在简陋的房间, 当: 开始写信, 那么: 回忆涌现
2. 场景设定: 信件写到一半, 当: 揭示秘密, 那么: 读者震撼

## 创作要求

- SR-001: 故事必须在5000字内完成
- SR-002: 必须有情感冲击力
- SR-003: 结局必须有反转

## 成功标准

- SC-001: 读者能在15分钟内读完
- SC-002: 90%的读者被结局震撼
```

#### 2. 创作计划

```markdown
# 创作计划: 最后一封信

## 创作背景

- 小说类型: 温情悬疑短篇
- 叙事视角: 第一人称(书信体)
- 文风特点: 温暖感人但暗藏悬疑
- 预计篇幅: 5000字

## 详细大纲

1. 开头(1000字): 老人开始写信,温暖的回忆
2. 发展(2000字): 通过信件展现爱情故事
3. 转折(1000字): 暗示老人的处境不寻常
4. 结局(1000字): 揭示老人在监狱,为保护妻子犯罪
```

#### 3. 任务列表

```markdown
# 创作任务: 最后一封信

- [ ] T001 撰写开头部分(1000字)
- [ ] T002 撰写发展部分(2000字)
- [ ] T003 撰写转折部分(1000字)
- [ ] T004 撰写结局部分(1000字)
- [ ] T005 整体润色和修改
- [ ] T006 质量检查
```

## 🔧 常见问题

### Q: 我必须使用所有模板吗?

**A**: 不必须。根据你的需要选择:
- 短篇: 可以只用规格说明和检查清单
- 长篇: 建议使用所有模板
- 团队: 强烈建议使用所有模板以统一标准

### Q: 这会限制我的创作自由吗?

**A**: 不会。这些模板是工具,不是枷锁:
- 你可以随时调整规格说明
- 可以根据灵感修改计划
- 可以自定义模板
- 重点是提供结构,而非限制创意

### Q: 适合什么类型的小说?

**A**: 几乎所有类型:
- 都市言情 ✅
- 玄幻修仙 ✅
- 科幻悬疑 ✅
- 历史架空 ✅
- 现实主义 ✅

关键是根据类型调整检查清单和创作原则。

### Q: 需要什么工具?

**A**: 非常简单:
- 文本编辑器(任何都可以)
- Markdown 支持(可选,但推荐)
- 版本控制(Git,可选但推荐)
- AI 助手(可选,但能大大提高效率)
